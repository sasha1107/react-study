# 컴포넌트 Importing and Exporting

컴포넌트의 마법은 재사용가능성(reusability)에 있다.

많은 컴포넌트를 중첩함에 따라 파일 분리의 필요성이 생긴다.

## root 컴포넌트

-   CRA로 생성된 프로젝트의 경우 일반적으로 `App.js`
-   Next.js와 같은 파일 기반 라우팅이 있는 프레임워크를 사용하는 경우 루트 컴포넌트는 페이지마다 다르다.

---

## Exporting and importing a component

하나의 파일에서 여러 개의 컴포넌트를 선언할 수 있지만, 큰 파일은 탐색하기 어려울 수 있다.

이를 위해 컴포넌트를 다른 파일로 분리할 수 있여 `export` / `import` 해 사용할 수 있다.

### export {} vs export default

        </td>
    </tr>
    <tr>

        </td>
        <td>
            import 할 때 중괄호 필요 X
        </td>
    </tr>
    <tr>
        <td>
            `export function Button() {}`
        </td>
        <td>
            `export default function Button() {}`
        </td>
    </tr>
    <tr>
        <td>
            `import { Button } from './Button.js';`
        </td>
        <td>
            `import Button from './Button.js';`
        </td>

    <tr>
        <td>

        </td>
        <td>

        </td>
    </tr>
</tbody>
</table>

> 파일은 하나의 default export 만 가질 수 있고, named export는 여러 개 가능하다

> `default export` / `named export` 중 알맞은 걸 선택하여 통일되게 사용할 것을 권장!

### 확장자 생략

다음과 같이 import 할 때 파일 확장자(`.js`)가 나타나지 않을 수 있다.

```
import Gallery from './Gallery';
```

확장자가 없어도 리액트에서 잘 작동한다.
[참고](https://developer.mozilla.org/ko/docs/Web/JavaScript/Guide/Modules)
