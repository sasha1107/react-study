리액트 애플리케이션은 `root` 컴포넌트에서 시작된다.
`root` 컴포넌트는 일반적으로 새로운 프로젝트를 시작할 때 자동적으로 생성된다.


- 예를 들어 CRA로 프로젝트를 생성하면 root 컴포넌트는 `src/App.js`이다.
- Next.js 프레임워크를 사용하는 경우 루트 구성요소는 `pages/index.js`애 정외된다. (13버전에서는 아마 `app/page.js`..)

대부분의 리액트 앱은 컴포넌트를 끝까지 사용한다.(all the way down)
> (small) 버튼 같은 재사용 가능한 부분에서부터 --> 전체 페이지까지 (large)

[리액트 기반의 프레임워크(Next.js, Remix, Gatsby 등)](https://react.dev/learn/start-a-new-react-project)는 빈 HTML 파일을 사용하고 JS로 페이지를 관리하도록 하는 대신, React 컴포넌트로부터 자동으로 HTML 파일을 생성한다.
-> 자바스크립트 코드가 로드되기 전에 앱에서 일부 콘텐츠를 표시할 수 있다.
